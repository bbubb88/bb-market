# 竞争对手支付方案调研

## 国内平台（IGXE 等）

### 支付方式
| 方式 | 说明 |
|------|------|
| 支付宝 | 0.6% 手续费 |
| 微信支付 | 0.6% 手续费 |
| QQ支付 | 0.6% 手续费 |
| 银行卡 | 1% 手续费 |
| USDT | 需商家入驻 |

### 特点
- 有**商家入驻**模式
- 买家付款到平台 → 确认后放款给商家
- 平台抽取**5-10%** 服务费

---

## 零成本、低风险方案

### 方案一：纯 C2C（最简单）

```
买家 ←→ 卖家直接交易
平台只收手续费（3-5%）
```

**优点**：零成本、零风险
**缺点**：无托管，信任度低

---

### 方案二：担保交易（人工）

```
1. 买家付款到平台钱包
2. 卖家交付账号
3. 买家确认 → 平台放款给卖家
4. 有纠纷 → 人工客服处理
```

**优点**：有保障、风险可控
**缺点**：需要人工审核

---

### 方案三：USDT 自动托管（推荐）

使用 **TronGrid** 免费 API 监听转账

```
1. 生成临时钱包地址
2. 买家转账到临时地址
3. 自动监听确认 → 通知平台
4. 确认后放款
```

**优点**：自动化、低手续费(~1%)
**缺点**：需要技术对接

---

## 最低成本方案建议

| 阶段 | 方案 | 成本 |
|------|------|------|
| **初期** | 人工担保 + 银行卡转账 | 1% 手续费 |
| **中期** | + USDT 人工确认 | ~1% 手续费 |
| **后期** | USDT 自动托管 | ~1% 手续费 |

### 建议初期做法
1. **银行卡转账** - 买家转账截图，人工确认
2. **支付宝/微信** - 有企业后接入
3. **USDT** - 统一地址，人工确认

这样**零技术成本**，只需人工审核！
